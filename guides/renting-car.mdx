## Overview

The SocialiCar renting system allows users to browse available vehicles, apply advanced filters, and make bookings. The platform prioritizes listings from premium subscribers and ensures vehicle availability through reservation conflict detection.

<Info>
  Available vehicles are displayed based on subscription tier: Premium listings appear first, followed by Plus, then standard listings.
</Info>

## Browsing Vehicles

### Search Functionality

The platform provides a comprehensive search system:

```php mostrar_coches.php
if (isset($_POST["buscar"])) {
    $buscar = mysqli_real_escape_string($_conexion, $_POST["buscar"]);

    $sql = mysqli_query($_conexion, "SELECT * FROM coche 
            WHERE 
                precio LIKE '%$buscar%' OR
                marca LIKE '%$buscar%' OR 
                modelo LIKE '%$buscar%' OR 
                ciudad LIKE '%$buscar%' OR 
                codigo_postal LIKE '%$buscar%' OR 
                color LIKE '%$buscar%' OR
                descripcion LIKE '%$buscar%' OR
                combustible LIKE '%$buscar%' OR
                transmision LIKE '%$buscar%' OR
                provincia LIKE '%$buscar%' OR
                tipo_aparcamiento LIKE '%$buscar%'");

    $numeroSql = mysqli_num_rows($sql);

    echo "<p class='text-success fw-bold mt-3'>
            <i class='mdi mdi-car-search'></i> $numeroSql resultados encontrados
        </p>";
}
```

**Searchable fields:**
- Price
- Brand and model
- Location (city, province, postal code)
- Color
- Description
- Fuel type
- Transmission type
- Parking type

## Vehicle Display Priority

SocialiCar displays vehicles based on subscription tiers to incentivize premium memberships:

### Premium Listings

```php mostrar_coches.php
$obtener_coche_premium = $_conexion->prepare("
    SELECT coche.*, sus.tipo AS tipo_suscripcion
    FROM coche
    JOIN usuario ON coche.id_usuario = usuario.identificacion
    JOIN suscripcion_usuario sus 
        ON sus.identificacion = usuario.identificacion 
        AND sus.activo = TRUE 
        AND sus.tipo = 'Premium'
    WHERE coche.provincia = ?
    AND NOT EXISTS (
        SELECT 1 FROM reserva_coche 
        WHERE reserva_coche.matricula = coche.matricula 
        AND (
            (? BETWEEN reserva_coche.fecha_inicio AND reserva_coche.fecha_final) OR
            (? BETWEEN reserva_coche.fecha_inicio AND reserva_coche.fecha_final) OR
            (reserva_coche.fecha_inicio BETWEEN ? AND ?) OR
            (reserva_coche.fecha_final BETWEEN ? AND ?)
        )
    )
    ORDER BY coche.precio ASC
    LIMIT 3
");
```

## Availability Checking

The platform automatically filters out vehicles with conflicting reservations:

<Warning>
  The reservation conflict detection ensures that vehicles already booked during the requested dates are not displayed in search results.
</Warning>

The SQL query checks for four types of date conflicts:
1. Start date falls within existing reservation
2. End date falls within existing reservation
3. Existing reservation start falls within requested period
4. Existing reservation end falls within requested period
